*División del proyecto YAPar en Python*

*Decisión técnica general*

El proyecto se implementará en Python.

El comando principal del generador será:

python yapar.py parser.yalp -l lexer.yal -o theparser.py

El archivo .yalp tendrá una sección de tokens, luego el separador %%, y después la sección de producciones. Los tokens se declaran con %token, los tokens ignorados con IGNORE, y las producciones usan :, | y ;.

El parser generado también será Python:

python theparser.py input.txt

Restricción crítica: está prohibido usar librerías de expresiones regulares (import re u otras). Toda funcionalidad de reconocimiento de patrones debe implementarse con autómatas finitos o parsing manual por caracteres/strings. Penalización por incumplimiento: 0 puntos.

La interfaz gráfica es obligatoria. Penalización por incumplimiento: -3 puntos.

La estructura recomendada del proyecto sería:

proyecto_yapar/
  yapar.py
  grammar.py
  yalp_reader.py
  slr_generator.py
  lr0_visualizer.py
  parser_runtime.py
  code_generator.py
  lexer_adapter.py
  gui.py
  requirements.txt
  README.md
  examples/
    parser.yalp
    lexer.yal
    input_valid.txt
    input_syntax_error.txt
    input_lexical_error.txt
  tests/
    yalp/
    slr/
    runtime/
    integration/

Recomendación: usar Python 3.10+ o Python 3.11+. Para pruebas, usar pytest. Para GUI, usar tkinter (incluido en stdlib) o customtkinter. Para visualización del autómata, usar graphviz o matplotlib + networkx.


*Persona 1: lector YAPar, modelo de gramática y validación*

*Objetivo*

Esta persona produce el objeto Grammar. No usa librerías de regex.

*Tareas específicas*

*1. Crear las estructuras base en Python*

En grammar.py, definir estructuras como:

from dataclasses import dataclass

@dataclass(frozen=True)
class Production:
    left: str
    right: tuple[str, ...]

@dataclass
class Grammar:
    tokens: set[str]
    ignore_tokens: set[str]
    productions: list[Production]
    start_symbol: str

Debe quedar claro que:

tokens = terminales declarados con %token
ignore_tokens = tokens declarados con IGNORE
productions = reglas de la gramática
start_symbol = primera producción encontrada


*2. Implementar el lector principal*

En yalp_reader.py, implementar:

    ...

Esta función debe:

procesar %token
procesar IGNORE
procesar producciones
validar errores
devolver Grammar

No se permite usar import re ni ninguna librería de expresiones regulares. Implementar con operaciones de strings (split, strip, find, startswith, etc.).


*3. Eliminar comentarios*

Debe eliminar comentarios delimitados por:

/* comentario */

También debe aceptar comentarios de varias líneas:

/*
  comentario
  de varias líneas
*/

Debe reportar error si hay comentario sin cerrar:

Error YAPar: comentario sin cerrar.


*4. Separar secciones usando %%*

Debe buscar exactamente:

%%

Reglas:

antes de %% = sección de tokens
después de %% = sección de producciones

Errores obligatorios:

Error YAPar: falta separador %% entre tokens y producciones.
Error YAPar: se encontró más de un separador %%.


*5. Procesar %token*

Debe aceptar:

%token TOKEN_1
%token TOKEN_2 TOKEN_3 TOKEN_4
%token NUMBER ID PLUS MINUS

Debe validar:

la línea inicia con %token
hay al menos un token después de %token
los tokens están en mayúscula (patrón: [A-Z][A-Z0-9_]*)
no hay tokens duplicados

Errores obligatorios:

Error YAPar: declaración %token sin tokens.
Error YAPar: token inválido 'token_1'. Los tokens deben estar en mayúscula.
Error YAPar: token duplicado 'ID'.


*6. Procesar IGNORE*

Debe aceptar:

IGNORE WS
IGNORE COMMENT
IGNORE WS COMMENT

Debe validar:

la línea inicia con IGNORE
hay al menos un token después de IGNORE
cada token ignorado fue declarado antes con %token

Errores obligatorios:

Error YAPar: declaración IGNORE sin tokens.
Error YAPar: token ignorado no declarado 'SPACE'.


*7. Procesar producciones*

Debe aceptar producciones multilínea:

expr:
    expr PLUS term
  | term
;

Y producciones de una línea:

factor: NUMBER | ID ;

Debe convertirlas a:

expr -> expr PLUS term
expr -> term
factor -> NUMBER
factor -> ID

Debe detectar:

nombre antes de :
alternativas separadas por |
fin de producción con ;
símbolos separados por espacios

Errores obligatorios:

Error YAPar: producción sin ':'.
Error YAPar: producción sin ';'.
Error YAPar: producción vacía.

Una producción vacía es cualquier alternativa que no contiene ningún símbolo, por ejemplo:

expr: ;          <- alternativa vacía
expr: PLUS | ;   <- segunda alternativa vacía


*8. Validar nombres*

Los no terminales deben iniciar en minúscula:

expr
term
factor
production1

Tokens/terminales deben seguir el patrón [A-Z][A-Z0-9_]*:

NUMBER
ID
PLUS
TOKEN_1

Errores obligatorios:

Error YAPar: nombre de producción inválido 'Expr'. Los no terminales deben iniciar en minúscula.
Error YAPar: nombre de producción inválido 'TOKEN'. Los no terminales deben escribirse como identificadores en minúscula.


*9. Validar símbolos usados*

En una regla como:

expr:
    expr PLUS term
;

Debe validar:

PLUS existe en tokens
expr y term existen como producciones

Errores obligatorios:

Error YAPar: token usado pero no declarado 'PLUS'.
Error YAPar: no terminal usado pero no definido 'term'.


*10. Definir símbolo inicial*

El símbolo inicial será la primera producción encontrada.

Ejemplo:

program:
    stmt_list
;

stmt_list:
    stmt
;

Entonces:

grammar.start_symbol = "program"
